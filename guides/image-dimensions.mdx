Passform automatically generates standard (@1x) and high-resolution (@2x) variants from a single upload. All images are converted to PNG during processing.

**Max file size:** 10 MB per image.

## Image types

<AccordionGroup>
  <Accordion title="Icon" icon="circle">
    Displayed in Apple Wallet as the small app-like icon on the pass.

    | Variant | Dimensions |
    |---------|-----------|
    | @1x | 29 × 29 px |
    | @2x (auto-generated) | 58 × 58 px |

    - **Shape:** Square
    - **Platforms:** Apple Wallet, Web
  </Accordion>

  <Accordion title="Logo" icon="image">
    Displayed at the top of a pass alongside the issuer name.

    | Variant | Dimensions |
    |---------|-----------|
    | @1x | 160 × 50 px |
    | @2x (auto-generated) | 320 × 100 px |

    - **Platforms:** Apple Wallet, Google Wallet
  </Accordion>

  <Accordion title="Banner" icon="panorama">
    The main hero image on the front of a pass. Apple Wallet calls this the "strip" image.

    | Variant | Dimensions |
    |---------|-----------|
    | @1x | 375 × 123 px |
    | @2x (auto-generated) | 750 × 246 px |

    - **Platforms:** Apple Wallet, Google Wallet
    - **Note:** Also used in link previews when sharing a pass URL.
  </Accordion>

  <Accordion title="Background" icon="desktop">
    Full-bleed background image shown on the web pass viewer. If not provided, `backgroundColor` is used instead.

    | Variant | Dimensions |
    |---------|-----------|
    | Standard | 1920 × 1080 px |

    - **Platform:** Web only
  </Accordion>

  <Accordion title="Thumbnail" icon="square-dashed">
    A square image displayed alongside the primary field on generic pass types. Replaces the banner in Apple Wallet when set.

    | Variant | Dimensions |
    |---------|-----------|
    | @1x | 90 × 90 px |
    | @2x (auto-generated) | 180 × 180 px |

    - **Platform:** Apple Wallet only
    - **Pass types:** `generic` only
  </Accordion>
</AccordionGroup>

## Uploading images

Upload images via the dashboard or the API. You only need to provide one file per image type — Passform generates the @2x variant automatically.

```bash
curl -X POST https://api.passform.io/v1/templates/{templateId}/images \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "icon=@icon.png" \
  -F "logo=@logo.png" \
  -F "banner=@banner.png"
```

The response includes URLs for each uploaded image:

```json
{
  "icon": "https://...",
  "logo": "https://...",
  "banner": "https://..."
}
```

Use these URLs in your template's `icon`, `logo`, and `banner` fields.

<Note>
  Google Wallet requires images to be publicly accessible. Images uploaded via the Passform API are hosted on public URLs and work automatically.
</Note>
